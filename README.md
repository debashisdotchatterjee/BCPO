# BCPO
This repository/script provides a lightweight validation environment for Bayesian Conservative Policy Optimization (BCPO) in an offline Reinforcement Learning (RL) setting.The script is designed to run in a Google Colab or local Jupyter environment. It attempts to load the standard D4RL continuous-control benchmarks. If D4RL is unavailable, it gracefully falls back to a custom synthetic tabular Gridworld environment to demonstrate the core mathematical concepts of the algorithm.

🚀 FeaturesD4RL Integration: Auto-detects the presence of d4rl and gym for continuous control experiments.Synthetic Gridworld Fallback: A fully customizable $n \times n$ stochastic Gridworld MDP with start, goal, and trap states to test offline RL agents when complex environments aren't available.Suboptimal Dataset Generation: Simulates an offline dataset by rolling out a noisy, $\epsilon$-greedy behavior policy.Automated Artifact Generation: Automatically packages learning curves, policy maps, state-value heatmaps, and performance summaries into a downloadable .zip file.

🧠 Algorithms Compared:  The synthetic fallback evaluates and compares three distinct approaches to offline RL:BCPO-lite (Ours): Uses Bayesian pessimism to penalize uncertain transitions and rewards based on state-action visitation counts, combined with KL-divergence regularization to anchor the learned policy to the behavior policy.Naive FQI (Fitted Q-Iteration): A standard Maximum Likelihood Estimation (MLE) approach that updates $Q(s,a)$ without any pessimism or safety constraints, making it susceptible to overestimating out-of-distribution (OOD) actions.Behavior Cloning (BC): A purely supervised baseline that directly mimics the empirical action distribution of the offline dataset.
